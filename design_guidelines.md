# 企业抽奖工具设计指南

## 1. 品牌定位

- **应用类型**: 企业内部抽奖工具
- **设计风格**: 简洁、专业、无干扰
- **目标用户**: 企业领导（抽奖操作）、员工（参与抽奖）
- **核心理念**: 界面干净无广告，让领导放心使用

## 2. 配色方案

### 主色调
- **主色**: `#1890ff` (蓝色) - 专业、信任
- **主色浅**: `#40a9ff`
- **主色深**: `#096dd9`

### 中性色
- **标题黑**: `#1a1a1a`
- **正文灰**: `#595959`
- **辅助灰**: `#8c8c8c`
- **边框灰**: `#d9d9d9`
- **背景灰**: `#f5f5f5`

### 语义色
- **一等奖金**: `#faad14` (金色)
- **二等奖银**: `#8c8c8c` (银色)
- **三等奖铜**: `#cd7f32` (铜色)
- **成功绿**: `#52c41a`
- **警告红**: `#ff4d4f`

## 3. 字体规范

- **H1 标题**: `text-3xl font-bold text-[#1a1a1a]`
- **H2 副标题**: `text-2xl font-semibold text-[#1a1a1a]`
- **H3 卡片标题**: `text-xl font-semibold text-[#1a1a1a]`
- **Body 正文**: `text-base text-[#595959]`
- **Caption 辅助**: `text-sm text-[#8c8c8c]`

## 4. 间距系统

- **页面边距**: `px-6 py-4`
- **卡片内边距**: `p-6`
- **组件间距**: `gap-4`
- **列表项间距**: `gap-3`

## 5. 组件使用原则

**必须优先使用 `@/components/ui/*` 的通用组件，禁止用 View/Text 手搓：**

- **按钮**: Button（主操作、次操作、危险操作）
- **输入框**: Input（姓名、工号）、Textarea（备注）
- **弹窗**: Dialog（确认、结果展示）、AlertDialog（危险操作确认）
- **提示**: Toast（操作反馈）、Sonner（轻提示）
- **卡片**: Card（奖项展示、中奖记录）
- **标签**: Badge（奖项等级、用户身份）
- **标签页**: Tabs（抽奖/管理切换）
- **滚动区域**: ScrollArea（长列表）
- **分割线**: Separator（内容分隔）
- **空状态**: 自定义空状态组件（无中奖记录时）

**容器样式**:
- 卡片圆角: `rounded-2xl`
- 按钮圆角: `rounded-xl`
- 输入框圆角: `rounded-xl`
- 阴影: `shadow-sm`

## 6. 导航结构

### 页面结构
```
├── pages/index/index.tsx          # 抽奖主页面
├── pages/login/index.tsx          # 登录页面
├── pages/admin/index.tsx          # 后台管理页面（仅领导）
└── pages/history/index.tsx        # 中奖记录页面
```

### TabBar 配置（可选）
如果需要 TabBar，配置如下：
- 抽奖（首页图标）
- 记录（列表图标）
- 管理（设置图标，仅领导可见）

## 7. 页面布局

### 登录页
- 顶部：Logo + 标题
- 中部：姓名输入框 + 工号输入框
- 底部：登录按钮
- 辅助：员工/领导身份选择（或自动识别）

### 抽奖主页
- 顶部：当前活动标题 + 用户信息
- 中部：奖项列表（一等奖、二等奖、三等奖）
- 核心区域：抽奖按钮（大按钮，带动画）
- 底部：中奖记录入口 + 管理入口（仅领导）

### 后台管理页（仅领导）
- 顶部：标题 + 新增奖项按钮
- 中部：奖项列表（可编辑、删除）
- 每项：奖项名称、等级、数量、操作按钮

## 8. 抽奖动画规范

### 抽奖按钮动画
1. **待机状态**: 脉动动画（scale 1.0 → 1.05 循环）
2. **抽奖中**: 旋转动画（rotate 360°）+ 倒计时数字
3. **结果展示**: 缩放弹跳（scale 0 → 1.2 → 1.0）

### 奖项滚动动画
- 横向滚动中奖者姓名
- 停留 3 秒后显示最终结果

### 中奖庆祝动画
- 礼花/彩带效果
- 中奖信息卡片淡入

## 9. 小程序约束

- **包体积**: 主包控制在 2MB 内
- **图片策略**: 所有图片资源使用 TOS 对象存储
- **性能优化**: 避免复杂动画导致卡顿，使用 CSS transform 而非 left/top
- **跨端兼容**: 确保微信小程序、H5 表现一致

## 10. 权限设计

### 用户角色
- **员工**: 可参与抽奖、查看自己的中奖记录
- **领导**: 员工权限 + 可管理奖项、查看所有中奖记录、新增/编辑/删除奖项

### 权限识别
- 通过工号前缀识别（如 L001 表示领导）
- 或通过预设白名单识别
